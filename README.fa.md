# One Chat

🌐 **زبان‌ها:** [English](README.md) | فارسی

One Chat یک رابط چت ساده و سبک برای صحبت با مدل‌های هوش مصنوعی است. هدف پروژه این است که برای استفاده از مدل‌های لوکال یا سرویس‌های مختلف، مجبور نباشید هر بار سراغ یک رابط جدا بروید؛ یک صفحه را باز می‌کنید، مدل را وصل می‌کنید و گفتگو را شروع می‌کنید.

کل برنامه داخل یک فایل HTML قرار دارد و برای اجرا به نصب، ساخت پروژه یا سرور جدا نیاز ندارد.

## چه مدل‌هایی را می‌شود وصل کرد؟

One Chat از چند سرویس آماده پشتیبانی می‌کند:

- Ollama
- LM Studio
- OpenRouter
- OpenAI
- Groq
- Together AI
- Fireworks AI
- DeepSeek
- Google Gemini
- Chrome AI یا Gemini Nano
- 9Router
- هر API سازگار با OpenAI

برای مدل‌های لوکال مثل Ollama و LM Studio معمولاً نیازی به API Key نیست. برای سرویس‌های آنلاین باید کلید همان سرویس را وارد کنید.

- امکان ارسال فایل‌های متنی، JSON، Markdown و تصویر
- ذخیره و جست‌وجوی گفتگوها
- خروجی گرفتن از گفتگو به‌صورت Markdown

- تم روشن و تاریک
- طراحی واکنش‌گرا برای دسکتاپ و موبایل

## راه‌اندازی

1. فایل `one-chat.html` را دانلود کنید.
2. فایل را با یک مرورگر جدید مثل Chrome، Edge یا Firefox باز کنید.
3. از بخش تنظیمات، Provider موردنظر را انتخاب کنید.
4. آدرس API، نام مدل و در صورت نیاز API Key را وارد کنید.
5. روی «آزمایش اتصال» بزنید و بعد گفتگو را شروع کنید.

همین؛ چیزی برای نصب یا Build کردن وجود ندارد.

## اتصال به Ollama

اول Ollama را اجرا کنید و مطمئن شوید مدل موردنظر روی سیستم شما آماده است. بعد در One Chat:

- Provider را روی `Ollama` بگذارید.
- آدرس پیش‌فرض را روی `http://localhost:11434/v1` نگه دارید.
- نام مدل نصب‌شده، مثلاً `llama3.2`، را وارد کنید.
- اتصال را آزمایش کنید.

اگر مرورگر اجازه اتصال نداد، باید دسترسی CORS را در تنظیمات Ollama برای آدرس صفحه فعال کنید.

## اتصال به LM Studio

در LM Studio سرور محلی را روشن کنید، سپس:

- Provider را روی `LM Studio` بگذارید.
- آدرس پیش‌فرض `http://localhost:1234/v1` را بررسی کنید.
- نام مدل لودشده را وارد کنید.
- روی «آزمایش اتصال» بزنید.

در صورت خطای اتصال، تنظیمات CORS سرور LM Studio را بررسی کنید.

## اتصال به APIهای دیگر

اگر سرویس شما API سازگار با OpenAI دارد، گزینه `OpenAI Compatible` را انتخاب کنید و این موارد را وارد کنید:

- آدرس پایه API، معمولاً آدرسی که به `/v1` ختم می‌شود
- نام دقیق مدل
- API Key، اگر سرویس به کلید نیاز دارد

از بخش «پارامترهای اضافی JSON» هم می‌توانید گزینه‌هایی مثل `top_p` را به درخواست اضافه کنید.

## حریم خصوصی و کلید API

گفتگوها و تنظیمات داخل مرورگر و روی همان دستگاه نگه‌داری می‌شوند و One Chat سرور واسطی برای دریافت پیام‌های شما ندارد. درخواست‌ها مستقیماً از مرورگر به Provider انتخاب‌شده ارسال می‌شوند.

اگر گزینه ذخیره کلید روی دستگاه خاموش باشد، API Key فقط تا زمان بسته شدن تب نگه‌داری می‌شود. اگر آن را روشن کنید، کلید در فضای ذخیره‌سازی مرورگر ثبت می‌شود؛ پس روی سیستم‌های عمومی یا مشترک این گزینه را فعال نکنید.

هنگام گرفتن فایل پشتیبان هم می‌توانید انتخاب کنید که کلیدهای API داخل فایل قرار بگیرند یا نه. برای امنیت بیشتر بهتر است کلیدها را وارد فایل پشتیبان نکنید.

## اگر اتصال برقرار نشد

این موارد را بررسی کنید:

- مدل یا سرور لوکال واقعاً در حال اجرا باشد.
- آدرس API و نام مدل درست وارد شده باشد.
- API Key معتبر باشد.
- مرورگر به آدرس مدل دسترسی داشته باشد.
- CORS روی سرور لوکال فعال باشد.
- اگر صفحه را مستقیم با `file://` باز کرده‌اید و Provider اتصال را مسدود می‌کند، فایل را با یک وب‌سرور ساده اجرا کنید.

---

One Chat قرار نیست جای ابزارهای سنگین و تخصصی را بگیرد؛ قرار است یک رابط دم‌دستی، مستقل و راحت باشد که سریع بتوانید مدل موردنظرتان را به آن وصل کنید و گفتگو را شروع کنید.
